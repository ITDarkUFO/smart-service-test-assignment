Логику, реализованную в данном наборе хранимых процедур, необходимо реализовать на С# в виде набора взаимодействующих компонентов. В рамках одного компонента необходимо реализовать логику, соответствующую одной схеме базы данных. При реализации необходимо учесть, что компоненты могут использоваться микросервисами и интегрироваться с базой данных с помощью ORM. Реализовать модульные тесты для одного выбранного компонента. 

Нужно исходить из того, что каждая схема БД, используемая в хранимых процедурах, реализует отдельную доменную модель. Соответственно, необходимо перенести реализацию каждой модели из хранимых процедур в отдельный компонент на C# с учетом того, что компонент может быть использован в микросервисной архитектуре.  Для тестового задания компоненты (классы) могут взаимодействовать напрямую через свои интерфейсы. При проектировании / реализации компонентов необходимо следовать SOLID принципам и продемонстрировать умение выделять уровни приложения.
